Programa¶

  • Algoritmo: a descrio de um padro de comportamento, especificado em termos de um conjunto bem definido e finito de aes primitivas que podem ser executadas.
  • Estrutura: de dados um modo particular de armazenamento e organizao de dados em um computador de modo que possam ser usados eficientemente.

Guimares, Lages. Algortimos e Estruturas de Dados, LTC, 1994

Programação Estruturada¶

  • Metodologia de projeto de programas que visa:
    1. Facilitar o desenvolvimento dos programas
    2. Facilitar a leitura (entendimento) dos programas
    3. Permitir a validao a priori dos programas
    4. Facilitar a manuteno e modificao dos programas

Guimares, Lages. Algortimos e Estruturas de Dados, LTC, 1994

Ciclo de Vida de um Software¶

1. Especificar os requisitos do problema

- Preparar uma especificao completa e no ambgua.

2. Analisar o problema

- Entender o problema.
- Avaliar solues alternativas.
- Escolher solues mais adequada.

3. Projetar o programa para solucionar o problema

- Fazer o projeto de cima para baixo (top-down) do sistema.
- Para cada mdulo, identificar as principais estruturas de dados e subprogramas associados.
- Desenvolver algoritmos e estruturas de dados dos subprogramas.

4. Implementar o projeto

- Codificar a soluo.
- Corrigir erros de codificao.

5. Testar e validar o programa

- Testar o cdigo e valid-lo, se correto.

6. Manter e atulalizar o programa

- Executar o sistema,
- Avaliar seu desempenho,
- Remover novos erros identificados, assim que detectados,
- Realizar modificaes de forma a manter o sistema atualizado,
- Validar as modificaes.

Koffman, Elliot B. Turbo Pascal 7.0, Addison-Wesley, 1994

Características da linguagem Python 3.x¶

  • Linguagem de programao de alto nvel
  • Multiparadigma (procedural, funcional e orientado a objetos )
  • Multiplataforma (Windows, Linux, iOS, etc.)
  • Interpretada (padro ou interativo)
  • Tipagem dinmica
  • Gerncia de memria automatizada(coletor de lixo)

Objetos¶

Todos dados em um programa Python so representados por objetos e pelas relaes entre objetos. Cada objeto no Python possui trs caractersticas:

  • Tipo (classe): nmeros, string, lista,etc.
  • Valor: Valores podem ser nmeros, palavras, etc.
  • Identidade: Parte da memria do computador que o objeto ocupa

A hierarquia conceitual do Python¶

De uma perspectiva mais concreta, os programas Python podem ser decompostos em mdulos, instrues, expresses e objetos, da seguinte maneira:

  1. Os programas so compostos de mdulos.
  2. Mdulos contm estruturas.
  3. Estruturas contm expresses.
  4. Expresses criam e processam objetos.

Range¶

O range representa uma sequncia imutvel de nmeros e comumente usado para repetir um nmero especfico de vezes em loops.

range(stop)

range(start,stop[, step])

  • start: O valor do parmetro inicio (ou 0 se o parmetro no foi fornecido)

  • stop: O valor do parmetro parada

  • step: O valor do parmetro passos (ou 1 se o parmetro no foi fornecido)

A vantagem do range em relao a uma lista ou tupla (veremos mais a diante) que um objeto range sempre ter a mesma quantidade (pequena) de memria, independentemente do tamanho do intervalo que representa (pois armazena apenas os valores de incio, parada e etapa , calculando itens e subfaixas individuais, conforme necessrio.

Função¶

Em termos simples, uma funo um dispositivo que agrupa um conjunto de instrues para que possam ser executadas mais de uma vez em um programa - um procedimento empacotado chamado pelo nome. As funes tambm podem calcular um valor de resultado e vamos especificar parmetros que servem como entradas de funo e podem diferir cada vez que o cdigo executado. Codificar uma operao como uma funo a torna uma ferramenta geralmente til, que podemos usar em vrios contextos.

  • Maximizando a reutilizao de cdigo e minimizando a redundncia
  • Decomposio processual

Escopo¶

Quando voc usa um nome em um programa, o Python cria, altera ou consulta o nome no que conhecido como namespace - um local onde os nomes residem. Quando falamos sobre a pesquisa do valor de um nome em relao ao cdigo, o termo escopo refere-se a um namespace: ou seja, o local da atribuio de um nome em seu cdigo-fonte determina o escopo da visibilidade do nome em seu cdigo.

Sendo assim, uma mesma variavl pode se referir a diferentes objetos, em um mesmo cdigo mas esses objetos tem que estar em escopos diferentes. Em Python temos o total de 4 escopos, local (L), enclosing function (E), global (G) e built-in (B), formando a regra LEGB.

Pilha de ativação¶

Quando uma funo chamada, duas tarefas so executadas:

  • Criao de um espao de memria para as variveis locais e parmetros;
  • Passagem efetiva de parmetros.

Durante a execuo de um programa uma rea especial da memria organizada em forma de pilha (stack). Ela utilizada para armazenar os valores das variveis locais e parmetros das funes. Outra rea organizada para manter as variveis globais do programa principal.

Quando o programa principal iniciado, um espao criado para manter as variveis globais. Sempre que uma funo ativada, um espao contendo os valores das variveis locais e dos parmetros reservado no topo da pilha. Ao fim de uma funo, seu espao automaticamente eliminado do topo da pilha, podendo ser reutilizado.

Arquivos de texto¶

Um arquivo texto uma sequncia de caracteres, organizada em linhas, que reside em uma rea de armazenamento (e.g., disco rgido, pen drive, CD/DVD) sob nome. Ns s trabalharemos com arquivos textos do tipo ".txt", bastante usado no armazenamento de dados. Outro tipo de arquivo texto muito utilizado o ".csv", aqueles que iro trabalhar com muitos dados frenquentemente usaro esse tipo de arquivo texto.

Veremos algumas funes que ns permite trabalhar com esses arquivos texto ".txt", sej, ler, editar ou criar

Abrindo arquivo texto¶

dados = open(teste.txt, r)

  • Caso o arquivo exista: Abre o arquivo teste.txt no modo leitura. A leitura comeara sobre o primeiro caractere da primeira linha do arquivo.
  • Caso ele no exista: causa erro FileNotFoundError.

dados = open(teste.txt, w)

  • Caso o arquivo exista: Apaga o contedo atual do arquivo "teste.txt" e abre o arquivo no modo escrita. A escrita comeara no incio do arquivo.
  • Caso ele no exista: Cria o arquivo "teste.txt" no diretrio e abre o arquivo no modo escrita. A escrita comeara no incio do arquivo.

dados = open(teste.txt, a)

  • Caso o arquivo exista: Abre o arquivo "teste.txt" no modo anexo sem apagar o contedo atual. A escrita comeara ao final do arquivo. Isto : Podemos anexar novas informaes ao arquivo.
  • Caso ele no exista: Cria o arquivo "teste.txt" no diretrio e abre o arquivo no modo anexo. A escrita comeara no final do arquivo que no caso o incio do arquivo.

Conjunto¶

Um conjunto (set) uma coleo no ordenada de objetos distintos imutvel(veis), tais como nmeros inteiros e de ponto flutuante, Strings e Tuplas. Os usos comuns incluem teste de associao, remoo de duplicatas de uma sequncia e computao de operaes matemticas como interseo, unio, diferena e diferena simtrica.

Como outras colees, os conjuntos suportam x in set, len (set) e for x in set. Sendo uma coleo no ordenada, os conjuntos no registram a posio do elemento ou a ordem de insero. Consequentemente, os conjuntos no suportam indexao, fatiamento ou outro comportamento semelhante a sequncia.

Atualmente, existem dois tipos de conjuntos build-in, set e frozenset. O tipo de conjunto set mutvel - o contedo pode ser alterado usando mtodos como add() e remove(). O tipo frozenset imutvel - seu contedo no pode ser alterado aps a criao.
